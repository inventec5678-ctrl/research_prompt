## ⚠️ 核心限制

### 明確定義「同一策略」的範圍

以下都算**同一策略**（不能一直調整）：
- RSI < 25 + MACD hist > 0
- RSI < 30 + MACD hist > 0  ← 這只是參數調整！
- RSI < 35 + MACD hist > 0  ← 這只是參數調整！

**定義**：一個策略 = 一組指標組合
- 這組組合**最多測試 3 次**
- 超過就**必須換完全不同的策略邏輯**

**錯誤示範（不要這樣做）**：
```
輪 1: RSI < 25 + MACD hist > 0
輪 2: RSI < 30 + MACD hist > 0  ← 只是改 RSI 閾值，仍然是同一策略
輪 3: RSI < 35 + MACD hist > 0  ← 又改 RSI，又只是參數調整
```

**正確示範**：
```
輪 1: RSI < 25 + MACD hist > 0 → 測試
輪 2: RSI < 25 + MACD hist > 0 + 1H 方向 → 這是新的策略！
輪 3: RSI < 25 + MACD hist > 0 + ATR 動態止損 → 這是新的策略！
```

### 限制：單一指標閾值調整不能超過 15 次

在 20 輪裡，以下調整**加起來不能超過 15 次**：
- RSI 超買/超賣值調整（30→35→40）
- MACD 閾值調整（0→-0.5→0.5）
- 其他單一指標準值的調整

**目的**：避免一直在同一個指標上調參數

## ⚠️ 核心邏輯：策略疊加（不是一直換策略！）

### 原則：失敗了要疊加，不是換掉

**錯誤做法（一直換策略）**：
```
輪 1: RSI + MACD → WR=47%（失敗）
輪 2: BB + RSI → （換全新的策略）
輪 3: ATR + BB → （又換全新的策略）
... 20 輪都是獨立測試，沒有累積！
```

**正確做法（策略疊加）**：
```
輪 1: RSI + MACD → WR=47%（基礎策略）
輪 2: RSI + MACD + 1H 方向 → WR=52%（進步！保留基礎，疊加新元素）
輪 3: RSI + MACD + 1H 方向 + Volume → WR=55%（繼續疊加！）
輪 4: RSI + MACD + 1H 方向 + Volume + ATR → WR=58%（繼續疊加！）
...
```

### 什麼時候「疊加」，什麼時候「換策略」？

| 情況 | 動作 |
|------|------|
| 策略有進步（WR 提升）| **疊加**：保留基礎，加入新元素 |
| 策略連續 3 輪無進步 | **換策略**：回到起點，重新開始 |
| 策略 WR < 40% | **換策略**：方向明顯錯誤 |

### 疊加的順序建議

當找到一個有潛力的策略時，按順序疊加：
1. **時間框架過濾**（1H 方向）→ 最有效
2. **趨勢指標**（MA/EMA）
3. **動量指標**（MACD/RSI）
4. **成交量確認**（Volume spike）
5. **波動率過濾**（ATR）

### 過濾器疊加方法（重要！）

每個過濾器都有 trade-off：

| 過濾器 | WR 效果 | 月交易效果 |
|---------|---------|-----------|
| CCI | ↑ 提高 | ↓ 減少 |
| Volume | ↑ 提高 | ↓ 減少 |
| PricePos | ↓ 降低 | ↑ 增加 |

**疊加順序**：
1. 先測每個過濾器**單獨的效果**（了解 trade-off）
2. 從對 WR 幫助最大的開始疊加
3. 觀察月交易的變化
4. 如果月交易掉太多，考慮加入能**彌補月交易的過濾器**

**目標**：找到 WR 和月交易**同時達標**的組合

**例子**：
```
Base: WR=60.16%, 月交易=98.2（兩個都接近達標）

Base+CCI: WR=59.75% ❌, 月交易=91.7（WR 差一點）
Base+PricePos: WR=56.05% ❌, 月交易=168.3（月交易超標）
→ 嘗試 Base+CCI+PricePos：兩者平衡？

目標：WR ≥ 60% 且 月交易 ≥ 100
```

## ⚠️ 決策標準（量化）

### 什麼是「參數調整」？❌ 不要一直做這個！

以下這些**只是參數調整**，不是策略改變：
- 換 RSI 超買/超賣值（30→45、70→55）
- 換 SL/TP 數值（0.5%→0.6%）
- 換 max_holding_bars（5→7）
- 只改止損止盈比例

### 什麼是「策略改變」？✅ 這個才是大步前進！

以下這些**才是策略改變**：
- 加入新指標（MACD、MA、BB）
- 加入時間框架過濾（1H、4H 方向）
- 加入形態識別（Pin Bar、Engulfing）
- 加入波動率過濾（ATR、VIX）
- 改變出场條件（動態 SL/TP、分批獲利）
- 換不同的進場邏輯（均值回歸→趨勢跟隨）

**核心原則**：每個策略最多微調參數 2-3 輪，如果 WR 沒明顯提升，**必須做策略改變**，而不是繼續調參數！

### 什麼時候換策略？

| 條件 | 動作 |
|------|------|
| **WR < 50%** | 果斷換策略 |
| **同一策略 3 輪調整都沒通過** | 換策略 |
| **連續 2 輪無進步** | 換策略 |
| **Monte Carlo 失敗** | 標記為失敗，換策略 |

### 鼓勵複雜策略！

**不要只做單一指標策略！** 實踐證明：
- 單一 RSI 策略有天花板（WR 上限 ~58%）
- 需要**多指標 Confluence** 才能突破

以下方向值得嘗試：
- **多指標 Confluence**：RSI + MACD + Volume 同時滿足
- **形態識別**：K線形態過濾（Pin Bar、Engulfing）
- **動態止損**：ATR-based SL/TP
- **分批獲利**：先平 50% → 移動 SL 到 breakeven
- **更長時間框架**：1H 或 4H 進場信號

### 什麼是「進步」？

| 指標 | 進步標準 |
|------|----------|
| WR | 提高 ≥ 2% |
| PF | 提高 ≥ 0.2 |
| Sharpe | 提高 ≥ 5 |

### 什麼時候繼續優化（已找到好策略）？

當策略**通過所有門檻**時：
- 繼續用這個策略測試不同參數組合
- 目標：找到更好的版本（更高的 Sharpe/PF）
- 確定真的無法再提升後，再換新策略探索
