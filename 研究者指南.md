# 研究者指南

## 你的任務

找到一個能通過以下**全部門檻**的交易策略：

| 指標 | 門檻 |
|------|------|
| WR（勝率） | ≥ 60% |
| PF（盈虧比） | ≥ 1.8 |
| DD（最大回撤） | ≤ 25% |
| Sharpe | ≥ 1.5 |
| 月交易筆數 | ≥ 100 |

## 你的工作

**只需要實現一個函數**：產生交易信號

```python
def generate_signals(data):
    """
    根據數據產生交易信號
    
    返回：1（做多）, -1（做空）, 0（不交易）
    """
```

**其餘全部由系統處理**：
- 回測引擎自動運行
- 進場/止損/止盈/出場全部由引擎控制
- 成本計算、指標計算都不需要你管

## 回測引擎參數

你只需要決定這些參數：

| 參數 | 說明 | 範例 |
|------|------|------|
| `stop_loss_pct` | 止損（相對於進場價） | `0.02` = 2% |
| `take_profit_pct` | 止盈（相對於進場價） | `0.04` = 4% |
| `max_holding_bars` | 最大持倉K線數（可選） | `None` = 不限制 |

## 程序（每次迭代）

### 步驟 1：設計進場邏輯

想想：
- 什麼情況下應該做多？
- 什麼情況下應該做空？
- 什麼情況下不交易？

**例子**：
- RSI 低於 30 → 做多
- RSI 高於 70 → 做空
- 其他 → 不交易

### 步驟 2：實現信號函數


## 分析與修正對照

看到結果後這樣調整：

| 結果顯示 | 問題 | 調整方向 |
|---------|------|----------|
| SL（止損）太多 | 止損太窄 | 放寬 stop_loss_pct 或加入進場過濾 |
| TP（止盈）太少 | 價格到不了止盈 | 收緊止盈或用動態止盈 |
| TO（超時）太多 | 交易太久 | 縮短 max_holding_bars 或收緊進場條件 |
| WR太低 | 進場時機不好 | 加入更多過濾條件 |
| PF太低 | 虧的比賺的多 | 放寬止盈、收緊止損、或改善進場 |

1. **專注在進場邏輯** — 系統處理其餘
